# 🐧 Queue
SystemV message ra đời trước. Ra đời sau là posix.

## SystemV 
### System V message queue identifier
Dùng hàm mssget để lẫy mã định danh để sau gửi và nhận.
```c
#include <sys/types.h>
#include <sys/msg.h>

int msgget(key_t key, int msgflg);
```
#### Key
key_t key: (key_t: int)
Có 2 cách lấy là
**Cách 1:** sử dụng 1 số interger random: int
**Cách 2:** Sử dụng hàm `flock()`
```c
#include <sys/ipc.h>
key_t ftok(char *pathname, int proj);
```
**Example:** 
```c
key_t key = ftok("myfile.txt", 65);
```
#### msgflg
message flag kết hợp mode|permission.
- permisson là kiểu giống quyền đọc ghi 666, 777 kiểu vậy.
- mode: gồm IPC_CREAT
**Example:**
```c
msgget(key, IPC_CREATE | 0666);
msgget(key, IPC_CREATE | IPC_EXCL | 0666);
```
#### Struct của message
Gồm mtype và mtext.
**Example:**
```c
struct msgbuf {
  long mtype;
  char mtext[];
}
```
### Send
```c
#include <sys/types.h>
#include <sys/msg.h>

int msgsnd(int msquid, const void *msgp, size_t msgsz, int msgflg);
```
### Received
```c
#include <sys/types.h>
#include <sys/msg.h>

ssize_t msgrcv(int msgpid, void *msgp, size_t maxmsgsz, long msgtyp, int msgflg);
```
Dùng lệnh sau để kiểm tra kích thước tôi đã của message có thể chứa.
```bash
cat /proc/sys/kernel/msgmax
```
msgtyp > 0: Nhận message đầu tiên có mtype == msgtyp
msgtyp = 0: Nhận msg ra đầu tiên không quan tâm đến msgtype
msgtyp < 0: nhận message có mtype nhỏ nhất mà mtype <= abs(msgtyp)
