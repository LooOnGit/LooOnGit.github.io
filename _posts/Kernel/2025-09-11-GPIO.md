## Kernel làm việc với GPIO
![alt text](/assets/BeagleBone/GPIO/image_1.png)

Không thể ghi trực tiếp vào **address physiscal** mà phải thông qua kernel address space có hàm `remap`.


Lưu ý điều này 1 word của 1 chip 32bit bằng 4byte, cho nên:
**Ví dụ:**
```c
(0x00 + 1) = 0x04 
```
Bởi vì 1 lần nhảy là `4byte` nên cần trọng có dùng `offset/4`.

## Code Led chớp tắt
file.h
```c
// gpio.h
#ifndef __LED_MODULE_H__
#define __LED_MODULE_H__

#define GPIO0_ADDR_BASE     0x44E07000
#define GPIO0_ADDR_END      0x44E07FFF
#define GPIO0_ADDR_SIZE     (GPIO0_ADDR_END - GPIO0_ADDR_BASE)

#define GPIO_OE_OFFSET			    0x134
#define GPIO_CLEARDATAOUT_OFFSET	0x190
#define GPIO_SETDATAOUT_OFFSET		0x194

#define GPIO0_30                    (1 << 30)       /* P9_11 */

#endif  /* __LED_MODULE_H__ */
```
file.c
```c
#include <linux/module.h>   /* This module defines functions such as module_init/module_exit */
#include <linux/io.h>       /* This module defines functions such as ioremap/iounmap */
#include "gpio.h"           /* Gpio modules */

#define DRIVER_AUTHOR "Loo"
#define DRIVER_DESC   "Turn on and off led"

uint32_t __iomem *gpio0_base_addr;

/* Constructor */
static int __init gpio_init(void)
{
    gpio0_base_addr = ioremap(GPIO0_ADDR_BASE, GPIO0_ADDR_SIZE);
    if(!gpio0_base_addr)
        return -ENOMEM;
    
    /* Set gpio0_30 as output mode and turn on */
    *(gpio0_base_addr + GPIO_OE_OFFSET / 4) &= ~GPIO0_30;
    *(gpio0_base_addr + GPIO_SETDATAOUT_OFFSET / 4) |= GPIO0_30;

    pr_info("Hello! Initizliaze successfully!\n");
    return 0;
}

/* Destructor */
static void __exit gpio_exit(void)
{
    *(gpio0_base_addr + GPIO_CLEARDATAOUT_OFFSET / 4) |= GPIO0_30;
    iounmap(gpio0_base_addr);

    pr_info("Good bye my fen !!!\n");
}

module_init(gpio_init);
module_exit(gpio_exit);

MODULE_LICENSE("GPL");
MODULE_AUTHOR(DRIVER_AUTHOR);
MODULE_DESCRIPTION(DRIVER_DESC); 
MODULE_VERSION("1.0"); 
```

## API GPIO Subsystem

### Interger based
- Phương pháp sử dụng số nguyên điều khiển GPIO.

### Cách dùng
```c
#define GPIO0_30            30

#define LOW                 0
#define HIGH                1

#define DRIVER_AUTHOR "phonglt15 p.linuxfromscratch@gmail.com"
#define DRIVER_DESC   "gpio subsystem"

/* Constructor */
static int __init gpio_init(void)
{
    gpio_request(GPIO0_30, "gpio0_30");
    gpio_direction_output(GPIO0_30, LOW);
    gpio_set_value(GPIO0_30, HIGH);

    pr_info("Hello! gpio status: %d!\n", gpio_get_value(GPIO0_30));
    return 0;
}

/* Destructor */
static void __exit gpio_exit(void)
{
    gpio_set_value(GPIO0_30, LOW);
    gpio_free(GPIO0_30);
    pr_info("Good bye my fen !!!\n");
}
```
`gpio_request(GPIO0_30, "gpio0_30")`

Yêu cầu quyền sử dụng số GPIO GPIO0_30 từ subsystem GPIO. Phải set gpio để tránh trường hợp 1 chân có nhiều chức năng. 

`gpio_direction_output(GPIO0_30, LOW)`

Thiết lập hướng của chân GPIO là output và đặt giá trị ban đầu bằng LOW (ở đây LOW = 0).

`gpio_set_value(GPIO0_30, HIGH)`

Ghi mức logic HIGH (1) lên chân GPIO đã được cấu hình output. Ở ví dụ này code set HIGH ngay sau khi direction_output đặt LOW — thứ tự này nghĩa là ban đầu direction set LOW, rồi lập tức set HIGH; có thể gộp hoặc tuỳ mục đích.

`gpio_free` trả lại resource cho kernel, tránh rò rỉ tài nguyên hoặc xung đột số GPIO.

Để ghi vào thì dùng `echo`, đọc ra thì `cat`.

Các bước điều khiển chân GPIO **ví dụ** chân 31.
**1. Export chân GPIO.**
Kích hoạt chân GPIO để hệ thống có thể sử dụng:
```bash
echo 31 > /sys/class/gpio/export
```
**2. Set the pin direction (input or output)**
**Ví dụ:** đặt làm chân xuất (output):
```bash
echo out > /sys/class/gpio/gpio31/direction
```
hoặc input:
```bash
echo in > /sys/class/gpio/gpio31/direction
```
**3. If an output pin, set the level to low or high**
- Mức thấp (0V):
```bash
echo 0 > /sys/class/gpio/gpio31/value
```
- Mức cao (3.3V hoặc 5V tùy hệ thống):
```bash
echo 1 > /sys/class/gpio/gpio31/value
```
**4. If an input pin, read the pin's level (low or high)**
```bash
cat /sys/class/gpio/gpio31/value
```
Kết quả sẽ là 0 hoặc 1.
**5. When done, unexport the pin**
Giải phóng chân để hệ thống không còn sử dụng:
```bash
echo 31 > /sys/class/gpio/unexport
```

Dùng cách này để tracking được trạng thái 1 chân chẳng hạn kiểu vậy xem có connect chưa.

## Device tree 
Convert hết các địa chỉ thanh ghi thành device tree kiểu vậy. Cho nên khi thay đổi phần cứng thì subsystem gọi như cũ, subsystem lớp này tự đến device tree để biết thao tác trên địa chỉ nào thanh ghi nào.


Hình dung giống như ông thợ sữa ông nước đến nhà chỉ cần cầm theo hộp đồ nghề thì hộp đồ nghề đó là **subsystem** còn bản vẽ nhà giống bảng datasheet có mấy thanh ghi vậy.

Đầu tiên có là `.dts` -> `.dtc` -> `.dtb`
- **s**: source
- **c**: compiler
- **dtb**: binary hệ thống sẽ đọc file này xem thằng nào đã được dùng cái nào không.

Có thể xem `.dts` để viết driver.
