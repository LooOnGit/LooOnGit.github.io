# 🐧 I2C Driver

## The driver architecture
Mỗi I2C được đại diện trong kernel dưới dạng instance `struct i2c_driver`.
### The I2C_driver structure
```c
struct i2c_driver {
    /* Standard driver model interfaces */
    int (*probe)(struct i2c_client *, const struct i2c_device_id *);
    int (*remove)(struct i2c_client *);
    /* driver model interfaces that don't relate to enumeration */
    void (*shutdown)(struct i2c_client *);
    struct device_driver driver;
    const struct i2c_device_id *id_table;
}
```
- `struct i2c_client` chứa các thông tin đặt thù của device, ví dụ: địa chỉ I2C của device.
### The probe() function
Hàm `probe()` là một phần của `struct i2c_driver`, và nó được thực thi bất cứ khi nào một thiết bị I2C (mà driver hỗ trợ) được khởi tạo instance trên hệ thống. Nó chịu trách nhiệm có các tác vụ:
- Kiểm tra liệu device có đúng là thiết bị mà driver hỗ trợ hay không.
- Kiểm tra xem I2C bus controller của SOC support các chức năng của yêu cầu hay không, bằng cách sử dùng `i2c_check_functionality()`.
- Khởi tạo thiết bị.
- Set up device-specific data.
- Register vào kernel framework phù hợp.


```c
static int foo_probe(struct i2c_client *client, const struct i2c_device_id *id);
```
- `struct i2c_client`: đại diện cho chính device i2c đó. Được define `include/linux/i2c.h`. Được define như sau:
```c
struct i2c_client {
    unsigned short flags; /* div., see below */
    unsigned short addr; /* chip address - NOTE: 7 bit */
    /* addresses are stored in the */
    /* _LOWER_ 7 bits */
    char name[I2C_NAME_SIZE];
    struct i2c_adapter *adapter; /* the adapter we sit on */
    struct device dev; /* the device structure */
    intirq; /* IRQ issued by device */
    struct list_head detected;
    #if IS_ENABLED(CONFIG_I2C_SLAVE)
    i2c_slave_cb_t slave_cb; /* callback for slave mode */
    #endif
};
```
- `struct i2c_device_id`: Con trỏ này trỏ tới phần tử (entry) ID của thiết bị I2C đã khớp với thiết bị đang được thăm dò (probe).
