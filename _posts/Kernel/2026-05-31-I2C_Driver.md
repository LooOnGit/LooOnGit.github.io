# 🐧 I2C Driver

## The driver architecture
Mỗi I2C được đại diện trong kernel dưới dạng instance `struct i2c_driver`.
### The I2C_driver structure
```c
struct i2c_driver {
    /* Standard driver model interfaces */
    int (*probe)(struct i2c_client *, const struct i2c_device_id *);
    int (*remove)(struct i2c_client *);
    /* driver model interfaces that don't relate to enumeration */
    void (*shutdown)(struct i2c_client *);
    struct device_driver driver;
    const struct i2c_device_id *id_table;
}
```
- `struct i2c_client` chứa các thông tin đặt thù của device, ví dụ: địa chỉ I2C của device.
### The probe() function
Hàm `probe()` là một phần của `struct i2c_driver`, và nó được thực thi bất cứ khi nào một thiết bị I2C (mà driver hỗ trợ) được khởi tạo instance trên hệ thống. Nó chịu trách nhiệm có các tác vụ:
- Kiểm tra liệu device có đúng là thiết bị mà driver hỗ trợ hay không.
- Kiểm tra xem I2C bus controller của SOC support các chức năng của yêu cầu hay không, bằng cách sử dùng `i2c_check_functionality()`.
- Khởi tạo thiết bị.
- Set up device-specific data.
- Register vào kernel framework phù hợp.


```c
static int foo_probe(struct i2c_client *client, const struct i2c_device_id *id);
```
- `struct i2c_client`: đại diện cho chính device i2c đó. Được define `include/linux/i2c.h`. Được define như sau:
```c
struct i2c_client {
    unsigned short flags; /* div., see below */
    unsigned short addr; /* chip address - NOTE: 7 bit */
    /* addresses are stored in the */
    /* _LOWER_ 7 bits */
    char name[I2C_NAME_SIZE];
    struct i2c_adapter *adapter; /* the adapter we sit on */
    struct device dev; /* the device structure */
    intirq; /* IRQ issued by device */
    struct list_head detected;
    #if IS_ENABLED(CONFIG_I2C_SLAVE)
    i2c_slave_cb_t slave_cb; /* callback for slave mode */
    #endif
};
```
- `struct i2c_device_id`: Con trỏ này trỏ tới phần tử (entry) ID của thiết bị I2C đã khớp với thiết bị đang được thăm dò (probe).

#### Per-device data
I2C cung cấp khả năng lưu trữ pointer tới bất kỳ data structure nào mà mình muốn sử dụng làm dữ liệu riêng cho thiết bị vào một `struct`.
```c
/* set the data */
void i2c_set_clientdata(struct i2c_client *client, void *data);
/* get the data */
void *i2c_get_clientdata(const struct i2c_client *client);
```
### The remove() function
```c
static int foo_remove(struct i2c_client *client);
```
## Driver initialization and registration
Thay vì dùng `init()` và `exit()`, thì nên dùng `module_*_driver()`. Điều này sẽ giúp driver chỉ load khi thực sự cần thiết, và unload khi không còn sử dụng.

```c
module_i2c_driver(foo_driver);
```
### Khởi tạo (provisioning) Driver và device
Trong matching mechanism, cần cung cấp một mảng `device_id` để khai báo những thiết bị mà driver hỗ trợ. Cấu trúc được sử dụng sẽ là `i2c_device_id`. Mảng này sẽ có nhiệm vụ thông báo cho Kernel biết driver này quan tâm đến những thiết bị nào.
```c
struct i2c_device_id {
    char name[I2C_NAME_SIZE];
    kernel_ulong_t driver_data; /* Data private to the driver */
};
```
Được define trong `include/linux/mod_devicetable.h`. Và `i2c_device_id` sẽ được embedded vào trong `struct i2c_driver`.


Ngoài ra, để cho I2C core (và cơ chế module auto-loading của udev) biết về danh sách các thiết bị mà ta hỗ trợ, chúng ta bắt buộc phải sử dụng macro `MODULE_DEVICE_TABLE`. Hơn nữa, Kernel cũng cần biết phải gọi hàm probe hay remove nào mỗi khi tìm thấy một thiết bị khớp ID trên thực tế. Đó là lý do tại sao các hàm `probe` và `remove` cũng phải được khai báo gộp chung vào trong cùng cấu trúc `i2c_driver` đó.
```c
static struct i2c_device_id foo_idtable[] = {
    { "foo", my_id_for_foo },
    { "bar", my_id_for_bar },
    { }
};
MODULE_DEVICE_TABLE(i2c, foo_idtable);

static struct i2c_driver foo_driver = {
    .driver = {
        .name = "foo",
    },
    .id_table = foo_idtable,
    .probe = foo_probe,
    .remove = foo_remove,
};
```
## Accessing the client
Các giao dịch trên bus nối tiếp (serial bus) thực chất chỉ là việc truy cập các thanh ghi (register) để đọc hoặc ghi dữ liệu.
- I2C cũng tuân theo nguyên tắc đó.
- I2C core cung cấp hai loại API:
    - API cho giao tiếp I2C thông thường (Plain I2C)
    - API cho các thiết bị tương thích SMBus
### Plain I2C communication
```c
int i2c_master_send(struct i2c_client *client, const char *buf, int count);
int i2c_master_recv(struct i2c_client *client, char *buf, int count);
```
- Hầu hết các hàm giao tiếp I2C đều nhận tham số đầu tiên là: `struct i2c_client`.
- Tham số thứ hai là vùng nhớ chứa dữ liệu cần đọc hoặc ghi.
- Tham số thứ ba là số byte cần đọc hoặc ghi.


Ngoài ra, cũng có thể thực hiện các giao dịch phức tạp hơn bằng:
```c
int i2c_transfer(struct i2c_adapter *adap, struct i2c_msg *msg,int num);
```
Trong đó `struct i2c_msg` được define như sau:
```c
struct i2c_msg {
    __u16 addr;   /* slave address */
    __u16 flags;  /* Message flags */
    __u16 len;    /* msg length */
    __u8 *buf;    /* pointer to msg data */
};
```
Trong đó:
- `addr`: Địa chỉ I2C của thiết bị.
- `flags`: Cờ để điều khiển giao tiếp I2C.
- `len`: Độ dài dữ liệu.
- `buf`: Con trỏ tới dữ liệu cần đọc hoặc ghi.


**Ví dụ**:
```c
ssize_t eep_read(struct file *filp, char __user *buf, size_t count, loff_t *f_pos)
```
